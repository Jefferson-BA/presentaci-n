Contexto

Problemática, Objetivos y Alcance

Problemática

Las MYPES de catering, como Hoja de Parra Spitz, enfrentan dificultades operativas por la ausencia de un sistema tecnológico integrado. La gestión dispersa en canales informales genera pérdida de información, errores manuales y mayores tiempos de respuesta.

Objetivos

Diseñar e implementar un sistema web a la medida para la gestión y automatización de pedidos, optimizando procesos operativos, centralizando la información comercial y mejorando la eficiencia en la atención al cliente.

Alcance

  • Módulo de gestión de clientes (frontend público)
  • Panel de administración de ventas y pedidos
  • Mantenimiento de información y contenido (CRUD)
  • Reportes básicos de seguimiento del negocio

Especificaciones

Requerimientos Funcionales Principales

Módulo del Cliente (Portal Público)

  • RF-01 Autenticación: registro e inicio de sesión seguro
  • RF-02 Catálogo interactivo: buffets árabe, parrillero, criollo
  • RF-03 Solicitud de pedidos: cotización con fecha, invitados y platos
  • RF-04 Trazabilidad: estado en tiempo real (Pendiente / Aprobado)

Módulo de Administración (Panel de Ventas)

  • RF-05 Gestión de pedidos: aprobación y actualización de estados
  • RF-06 Mantenedor de carta (CRUD): platos, buffets y precios
  • RF-07 Agenda de eventos: calendario anti-cruce de fechas
  • RF-08 Reportes y KPIs: ventas, pedidos y alta demanda

Super Administrador (Soporte TI)

  • RF-09 Control de accesos: roles y permisos
  • RF-10 Mantenimiento de datos: control y auditoría PostgreSQL

Negocio

Modelo Empresarial / Valor Institucional

Valor para la Empresa

  • Centralización de productos, usuarios y pedidos
  • Automatización de compra y seguimiento de órdenes
  • Atención con asistente virtual IA
  • Control de acceso por roles y permisos
  • Mejor experiencia de compra para usuarios finales

Beneficios Operativos

  • Reducción del tiempo en gestión manual de pedidos
  • Mayor control de inventario y disponibilidad
  • Seguimiento en tiempo real de operaciones
  • Mejora en atención con respuestas automatizadas
  • Escalamiento para nuevos módulos y funcionalidades

Indicadores Sugeridos (KPIs)

Cierre

Conclusiones y Recomendaciones

Conclusiones

  • Se desarrolló una plataforma que optimiza la gestión de productos, usuarios y pedidos
  • React, NestJS y PostgreSQL permitieron un sistema moderno, seguro y escalable
  • La IA mejora la atención al cliente con respuestas automatizadas
  • Roles y permisos contribuyen a la seguridad y administración eficiente

Recomendaciones

  • Implementar nuevas medidas de seguridad para proteger la información
  • Incorporar reportes y estadísticas para la toma de decisiones
  • Mejorar continuamente el asistente virtual para mayor precisión
  • Realizar pruebas periódicas para garantizar el correcto funcionamiento
